Phase 1: Security Hardening (Week 1–2)

1.1 Migrate Dev Secrets to Key Vault

# Create Key Vault for dev
az keyvault create \
  --name recovery-kv-dev \
  --resource-group recovery-rg \
  --location uaenorth

# Enable CSI driver addon (if not enabled)
az aks enable-addons \
  --addons azure-keyvault-secrets-provider \
  --name Recovery-aks-dev \
  --resource-group recovery-rg

# Migrate each secret
az keyvault secret set --vault-name recovery-kv-dev --name MongoConnectionString --value "<value>"
az keyvault secret set --vault-name recovery-kv-dev --name RedisConnectionString --value "<value>"

1.2 Network Policies

# base/network-policies/deny-all.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all
spec:
  podSelector: {}
  policyTypes:
    - Ingress
    - Egress

---
# Allow ingress controller to reach pods
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-ingress-controller
spec:
  podSelector: {}
  ingress:
    - from:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: ingress-nginx
  policyTypes:
    - Ingress

1.3 Pod Security Standards

# Apply restricted PSS to prod namespace
kubectl label namespace prod \
  pod-security.kubernetes.io/enforce=restricted \
  pod-security.kubernetes.io/warn=restricted

Checklist:

  • ☐ Container images scanned (Trivy in CI)

Phase 2: Autoscaling & Resilience (Week 3–4)

2.1 Horizontal Pod Autoscaler

# overlays/prod/hpa-claims.yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: prod-claims-hpa
  namespace: prod
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: prod-claims
  minReplicas: 2
  maxReplicas: 5
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
    - type: Resource
      resource:
        name: memory
        target:
          type: Utilization
          averageUtilization: 80

2.2 Cluster Autoscaler

az aks update \
  --name Recovery-aks-prod \
  --resource-group Recovery-rg-prod \
  --enable-cluster-autoscaler \
  --min-count 2 \
  --max-count 5

2.3 Pod Disruption Budgets

Phase 4: GitOps Implementation (Week 7–8)

4.1 ArgoCD Installation

# Install ArgoCD
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# Create Application for prod
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: prod-services
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/Mosadad/kubernetes.git
    targetRevision: main
    path: overlays/prod
  destination:
    server: https://kubernetes.default.svc
    namespace: prod
  syncPolicy:
    automated:
      prune: true
      selfHeal: true

6.2 Pipeline Template

  push:
    branches: [dev, qa, staging, prod]
jobs:
  build-test:
    # Build + unit tests
  scan:
    # Trivy container scan
  deploy:
    needs: [build-test, scan]
    environment: ${{ github.ref_name }}  # Requires approval for prod
    # Deploy to AKS
  smoke-test:
    needs: deploy
    # Health check endpoints
  rollback:
    needs: smoke-test
    if: failure()
    # kubectl rollout undo

7.2 Automated Backups

# Cron job for daily backup to Azure Blob
0 2 * * * mongodump --uri="$MONGO_URI" --gzip --archive=/backup/daily-$(date +\%Y\%m\%d).gz
# Upload to blob storage
0 3 * * * az storage blob upload --account-name recoverystorageprod --container-name backups --file /backup/daily-*.gz

Checklist:

  • ☐ Connection strings updated to use replica set

Phase 8: Performance & Optimization (Week 17–20)

8.1 Load Testing

# k6 load test for claims API
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '2m', target: 50 },
    { duration: '5m', target: 100 },
    { duration: '2m', target: 0 },
  ],
};

export default function () {
  const res = http.get('https://api.recoveryclaims.ae/api/claims/dashboard');
  check(res, { 'status is 200': (r) => r.status === 200 });
  sleep(1);
}

  • Reserved instances for prod (1-year commitment = 30% savings)
  • Storage lifecycle policies (move cold data to archive tier)

Checklist:

  • ☐ Off-hours scheduling for dev/QA
  • ☐ Reserved instances evaluated
